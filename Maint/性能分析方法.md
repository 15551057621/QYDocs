# 性能分析方法

## 占用分析表

当服务器TPS降低时，及时分析性能占用表便会起到很大的作用！可以及时分析卡顿源头，不会摸不着头脑

### 命令解析 
  `/timing [分析刻时长: 数字]`

执行后会返回一系列占用内容，以下是常见内容解析
> 如果看不懂占用分析表，可以试着喂给AI

### 名词解析
  
| 占用项名词 | 解释 |
| :--------------- | :--------------------------------- |
| `ActorLegacyTickSystem` | 处理实体的基础逻辑（例如生物、掉落物等） |
| `LevelChunkTickingSystem` | 处理区块的加载和更新（例如地形、红石电路等） |
| `GoalSelectorSystem` | 控制生物的人工智能（例如僵尸追踪、村民路径寻找等） |
| `SendActorDirtyData` | 负责将实体数据同步到客户端 |
| `ActorMotionSyncSystem` | 负责同步实体的移动（例如生物行走、飞行等） |

### 命令解析
`/prof [normal|entity|chunk|pt] [分析刻时长: 数字]`
- `normal` - 以默认形式分析健康状态
- `entity` - 分析实体对于服务器的影响
- `chunk` - 分析区块对于服务器的影响
- `pt` - 分析计划刻对于服务器的影响

### 返回参数解析

| 命令 | 功能描述 |
| :--- | :--- |
| `entity` | 列出所有已加载的实体以及每个实体在服务器刻（Tick）中所占用的处理时间。 |
| `chunk` | 列出所有已加载的区块及每个区块在服务器刻（Tick）中所占用的处理时间。 |
| `pt` | 列出所有有计划刻（Pending Tick）的区块。计划刻用于处理如红石电路、作物生长等延迟发生的方块更新。 |
