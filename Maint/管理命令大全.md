# 管理命令大全

## 玩家聊天称号相关

### 给予玩家一个临时称号
`tag <选择器> add "usf.tag:<字符串: 称号>§r"`
> 用此方法给予的称号不会出现在 /chattag 列表内

### 给予玩家永久的称号
`tag <选择器> add "tag:<字符串: 称号>"`

### 移除玩家的单个永久称号
`tag <选择器> remove "tag:<字符串: 称号>"`

### 重置玩家当前佩戴的称号
`tag <选择器> add "usf.tag:Reset"`
> 此方法仅重置佩戴的称号，给予的永久称号不变

## 向后台输出一条日志(可在命令方块里用)
logger \<数字: 输出模式> \<字符串: 输出内容>

- 输出模式:
  - 0 - `info`
  - 1 - `warn`
  - 2 - `error`
  - 3 - `debug`

### 示例：
`/logger 1 test log`

## LeviLamina的主要命令

ll \<后续操作: 往下看>

- `enable <字符串: 插件文件夹名称>`       - 启用一个插件
- `disable  <字符串: 插件文件夹名称>`      - 禁用一个插件
- `show  <字符串: 插件文件夹名称>`         - 获取插件信息
- `reload  <字符串: 插件文件夹名称>`       - 重载一个插件
- `unload  <字符串: 插件文件夹名称>`       - 卸载一个插件
- `reactivate  <字符串: 插件文件夹名称>`   - 重新激活一个插件
- `load  <字符串: 插件文件夹名称>`         - 加载一个插件
- `list`                              - 获取所有已加载的插件名称

### 示例：

`/ll reload QYServer`

- 不要随意重载一个DLL插件，修改配置文件后建议使用 重新激活插件
- 不要随意重载一个DLL，某些插件重载后可能会导致崩溃或坏档
- 重载JS插件风险小，但不是没有风险，不要重载监听事件过多的插件

## 停止向存档写入数据 (⚠️操作不当会导致坏档)

`save <模式>`
  - `query`  查询是否正在写入数据
  - `hold`   停止写入数据
  - `resume` 恢复数据写入

### 示例：

`/save query`

- 注：可用于不停机备份存档（正常情况下建议关机备份）
- 注：不要长时间停止写入，备份完成后第一时间恢复数据写入
- ⚠️ 停止写入期间 如果有大量数据操作 恢复写入时会导致服务端假死
- ⚠️ 这种情况切记 不要 强制关机！！！轻则数据丢失，重则整个存档损坏

## NBT修改相关命令

`/nbt item`                                - 修改手持物品NBT
`/nbt entity <实体>`                        - 修改实体NBT
`/nbt block <坐标>`                         - 修改方块，方块实体，方块容器NBT
`/nbt get_nbt_stick`                        - 获取"NBT修改工具"
`/nbt collection`                           - 打开收藏夹
`/nbt collection item <收藏物品名> <玩家>`    - 快速将自己物品收藏夹中的NBT物品给予玩家
`/nbt collection block <收藏方块名> <坐标>`   - 快速放置自己方块收藏夹中的NBT方块
`/nbt collection entity <收藏实体名> <坐标>`  - 快速生成自己实体收藏夹中的NBT实体，部分实体类型无效
`/nbt reload`                               - 重加载数据与配置文件
